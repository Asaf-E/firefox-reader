Rewrite settings

This stores your endpoint, model, and key in local extension storage for personal use.

How rewrite setup works

The reader works even without rewrite settings. If you leave these fields empty, you can still use the extension for normal reading, bookmarks, resume tracking, and chapter continuation.

If you want rewriting, this extension sends the chapter text to an OpenAI-compatible /v1/chat/completions endpoint.

  • API endpoint: your chat completions URL, for example https://api.openai.com/v1/chat/completions.
  • Model: the model name your endpoint expects. For this project, qwen2.5-7b-instruct has been a good local baseline.
  • API key: needed for hosted APIs, and sometimes for local servers if you enabled auth.

For local rewriting, use a server that exposes an OpenAI-compatible chat endpoint. Common options are LM Studio, llama.cpp server, vLLM, or an OpenAI-compatible wrapper around your local model stack.

Suggested direction: use a faithful instruct model rather than a dramatic prose model. Based on our tests here, qwen2.5-7b-instruct has been a safer default than aya-expanse-8b for preserving chapter meaning.
